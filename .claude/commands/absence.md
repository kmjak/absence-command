公欠申請を半自動で行うSkillです。以下の手順に従って処理を進めてください。

---

## Step 1: 日付入力

以下を聞いてください:

```
公欠を申請する日付を入力してください（例: 2026-07-03）:
```

入力された日付の曜日を算出し、`data/weekdays.md` に記載されている曜日に含まれるか確認してください。

**含まれない場合**は「その日は授業がないため申請できません」と伝えて処理を終了してください。

**含まれる場合**は `YYYY-MM-DD (曜日)` の形式で確認を表示してから次のステップへ。

例: `2026-07-04 (土曜日)`

---

## Step 2: 授業選択

`data/timetable.md` を読み込み、入力された日付の曜日に対応する授業を全て取得してください。

**授業が存在しない曜日の場合**は「その曜日には授業がありません」と伝えて処理を終了してください。

授業が存在する場合は番号選択形式で提示し、公欠を申請する授業を複数選択させてください。

```
公欠を申請する授業を選んでください（複数選択可）:

  1) 1限/ネットワーク演習Ⅱ/山本先生
  2) 2限/ネットワーク演習Ⅱ/山本先生

番号をカンマ区切りで入力してください（例: 1,2）:
```

ユーザーが入力したら次のステップへ。

---

## Step 3: 公欠日付の特定

Step 1 で入力された日付をYYYYMMDD形式に変換して使用してください。

---

## Step 4: 写真の確認

以下を聞いてください:

```
公欠理由の証明写真はありますか？
写真がある場合はファイルパスを入力してください（なければEnterを押してください）:
```

---

## Step 5-A: 写真がない場合

公欠の理由をテキストで入力させてください:

```
公欠の理由を入力してください:
```

入力されたら **Step 6** へ進んでください。画像名と画像BASE64は空文字で渡してください。

---

## Step 5-B: 写真がある場合

以下を聞いてください:

```
公欠理由を…
  (1) 自分で入力する
  (2) Claudeに写真をもとに考えてもらう
```

### (1) 自分で入力する場合

公欠の理由をテキストで入力させてください。その後 **Step 6** へ。

### (2) Claudeに考えさせる場合

写真を読み込み、イベント・行事の内容を把握したうえで、以下のフォーマットで簡潔な公欠理由を生成してください:

```
「イベント名」に参加するためです。
（参加理由を1〜2文で。自分が何を知りたかった・解決したかった・学びたかったかを学生らしい口調で）
```

例:
```
「DBデザイン勉強会」に参加するためです。
DBのid管理方法をUUIDにするかintのauto incrementを使うかで悩んでいたので参加します。
```

生成した理由をユーザーに提示し、確認を取ってから **Step 6** へ。

---

## Step 6: run.sh の実行

`./scripts/submit.sh` と `./scripts/run.sh` の内容を確認したうえで、以下の形式で `./scripts/run.sh` を実行してください。

写真がある場合は事前に以下のコマンドでBASE64に変換してください（データURIプレフィックスつき・改行なし）:
```bash
IMAGE_BASE64="data:image/png;base64,$(base64 -i <写真のパス> | tr -d '\n')"
```

選択された授業を DATE/PERIODS のペアとして順番に渡してください（最大5件）。
授業が5件未満の場合、残りのDATE・PERIODSは空文字を渡してください。

PERIODSのフォーマット: `x限目/授業名/講師名先生`（data/timetable.mdの形式から生成）

```bash
./scripts/run.sh \
  "<理由>" \
  "<DATE1>" "<PERIODS1>" \
  "<DATE2>" "<PERIODS2>" \
  "<DATE3>" "<PERIODS3>" \
  "<DATE4>" "<PERIODS4>" \
  "<DATE5>" "<PERIODS5>" \
  "<IMAGE_NAME>" "<IMAGE_BASE64>"
```

授業が足りない枠は空文字 `""` で埋めてください。

---

## Step 7: ログの保存

`logs/log/` ディレクトリに以下の形式でログファイルを保存してください。

ファイル名: `YYYY-MM-DD-<title>.md`
- `YYYY-MM-DD` は公欠で休む日付
- `<title>` は公欠内容から適切に考えてください（例: `network-exercise`, `business-training`）

内容:

```markdown
# 公欠申請ログ

## 申請日時
<実行した日時>

## 公欠日
<YYYY-MM-DD>

## 対象授業
<選択した授業一覧>

## 公欠理由
<使用した理由>

## 証明写真
<写真ファイル名 or なし>

## 申請結果
<scripts/run.shの実行結果>
```

`logs/log/` ディレクトリが存在しない場合は作成してください。
